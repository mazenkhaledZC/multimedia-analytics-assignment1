Ingestion — PDF → Images

Each PDF is rendered at 150 DPI into JPEG page images using pdf2image. This preserves every visual element: text layout, figures, tables, equations.

pdf2image · 150 DPI · JPEG quality 85
🔬

ColPali — Visual Patch Embeddings

ColPali v1.2 (based on PaliGemma) splits each page into a grid of visual patches and embeds each patch into a 128-dimensional vector. A single page yields ~1,024 patch vectors — capturing local visual detail, not just a global summary.

vidore/colpali-v1.2 · bfloat16 · MPS
🗄️

Qdrant — MultiVector Index

All patch embeddings are stored in Qdrant's MultiVector collection. Each page is a multi-vector point: 1,354 pages × ~1,024 patches × 128 dims.

COLLECTION: nasa_colpali · Distance: COSINE · MultiVector MaxSim
🔍

Retrieval — MaxSim Late Interaction

At query time, the text query is embedded by ColPali into query-token vectors. Qdrant computes MaxSim — for each query token, find the most similar patch vector, then sum the maxima. This is the same late-interaction scoring as ColBERT, applied visually.

Score(q, d) = Σ max_j cos(qᵢ, dⱼ)
👁️

Generation — Qwen2-VL-2B

The top retrieved page images are fed alongside the question into Qwen2-VL-2B-Instruct, a 2B parameter vision-language model running fully locally on Apple MPS. The model reads the actual page image and produces a grounded answer.

Qwen/Qwen2-VL-2B-Instruct · bfloat16 · max_new_tokens=300

💡

No OCR = No Bottleneck

Treating pages as images removes the OCR step entirely. Charts, handwritten annotations, and complex layouts are handled natively by the vision encoder.

🎯

MaxSim Beats Cosine for Documents

Late-interaction MaxSim finds the right page even when only one patch region is relevant to the query — a single-vector embedding would dilute this signal.

⚡

bfloat16 is Critical on MPS

float16 causes NaN/inf during sampling on Apple MPS due to lower dynamic range. Switching to bfloat16 resolves numerical instability in the VLM.

🔗

Cross-Modal is the Hardest

Queries that require reading both a figure and a related table (e.g. "compare chart data with table values") are the most challenging for both retrieval and generation.

Real query types the system handles — grounded in retrieved page images from NASA technical reports.

What are the main objectives of this research?
Answer · Text Query
The research investigates propulsion efficiency in variable-thrust rocket nozzles under high-altitude conditions, focusing on combustion stability and structural thermal limits across a range of thrust levels.

What are the performance values in the results table?
Answer · Table Query
The results table on page 12 shows thrust efficiency ranging from 87.3% to 94.1% across five test configurations, with peak specific impulse of 312 s at a chamber pressure of 2,200 psi.
[nasa_19840023019, p.12]
What does the pressure vs velocity graph show?
Answer · Figure Query
The graph shows an inverse relationship between chamber pressure and exhaust velocity. As pressure increases from 500 to 3,000 psi, exit velocity peaks around 2,200 psi before declining due to nozzle choking effects.
[nasa_19850015958, p.8]
What is the maximum thrust efficiency and at what temperature?
Answer · Cross-Modal Query
Based on the data visible on this page, the maximum thrust efficiency of 94.1% is achieved at a combustion chamber temperature of approximately 3,450 K, as indicated in Table 3 alongside the efficiency curve in Figure 6.
[nasa_19870013576, p.15] · [nasa_19890009866, p.22]
